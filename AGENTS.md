# lpwallet — Agent Guide

Loyalty Points Wallet backend service for Sanlam SFTX. Tracks member account balances and loyalty point transactions.

---

## Project Facts

- **Module:** `github.com/warkanum/lpwallet`
- **Go version:** 1.26.4
- **ORM:** GORM with `database/sql` stdlib types
- **Database:** PostgreSQL (default with `DATABASE_URL`) or SQLite (default when no postgres config is present)
- **Default listen address:** `127.0.0.1:8700`
- **HTTP library:** standard `net/http` (no framework)
- **YAML:** `gopkg.in/yaml.v3`

## Database-First Workflow

**Always edit the DBML first**, then regenerate models, then update handlers.

Source file: `sql/model/database_model.dbml`

Regenerate Go models:
```bash
relspec convert --from dbml --from-path ./sql/model/database_model.dbml --to gorm --to-path ./internal/models --package models --types stdlib
```

Generated files follow the naming pattern `sql_public_<table>.go` and are marked `// Code generated by relspecgo. DO NOT EDIT.`

## Schema Notes

- **User roles:** `role` field is `"member"` or `"admin"`
- **Password:** SHA-256 hashed, stored in `user.password`
- **OAuth-style clients:** `user.client_id` / `user.client_secret` for machine auth
- **Sessions:** `user_session.authtoken` is the bearer token; check `expiresat` on every request
- **Account balance:** `account.balance numeric(16,2)` — recalculate from transactions on mutation
- **Transaction action:** `account_transaction.action` is `"spend"` or `"earn"`
- **Transaction uniqueness:** `(reference, rid_account)` is unique — idempotent capture
- **Audit trail:** every write operation (create/update/delete) on `user`, `account`, `account_transaction` must write to `audit_event` + `audit_detail` rows within the same GORM transaction

## API Design Rules

- All endpoints return JSON; `Content-Type: application/json`
- Auth check: `Authorization: Bearer <authtoken>` header; validate against `user_session` (not expired)
- Store authenticated user in `context.Context` so handlers can access it without re-querying
- Use proper HTTP status codes: `200`, `201`, `204`, `400`, `401`, `403`, `404`, `409`, `500`
- All DB writes use a GORM transaction (`db.Transaction(func(tx *gorm.DB) error { ... })`)
- Admin-only operations: delete user, delete account, access other member accounts.
- Error body: `{"error": "<message>", "code": "<short_code>"}`
- Prevent counting the same transaction more than once when the same reference appears again (Implement this in the gorm before transactions)
- Do not allow a spend that would drive the balance below zero (Implement this in the gorm before transactions)

### CSV Batch Ingestion (`POST /api/v1/transactions/batch/csv`)

- `Content-Type: text/csv`
- First row must be the header: `ref,account_id,kind,points,occurred_at`
- `account_id` is the integer `id_account`
- `kind` maps to `action` — must be `earn` or `spend`
- `points` maps to `amount` — parsed as float64
- `occurred_at` accepted formats: RFC3339, `2006-01-02T15:04:05`, `2006-01-02 15:04:05`, `2006-01-02`; defaults to NOW() if empty
- Rows are processed in order; first failure aborts with the row number in the error message
- Each row goes through the same `captureTransaction` logic as the single and JSON batch endpoints (duplicate check, balance check, audit trail)
- Returns `201` with the array of created transactions on full success

Example:
```csv
ref,account_id,kind,points,occurred_at
tx-001,123,earn,150,2024-06-01T10:00:00
tx-002,123,spend,50,2024-06-02T09:00:00
```

## Authentication Flow

1. `POST /api/v1/auth/login` (public) — verifies `email` + `password` (SHA-256), creates a `user_session` row, returns `authtoken`
2. `POST /api/v1/auth/logout` (auth required) — expires the session
3. Middleware reads `Authorization: Bearer <token>`, looks up `user_session` where `authtoken = ? AND expiresat > NOW()`, loads user, stores in context

## Configuration

Config lives in `internal/config/config.go` as a plain struct loaded once at startup and injected everywhere — no global config variable.

### Config struct

```go
type Config struct {
    // HTTP
    ListenAddr string // host:port — env LISTEN_ADDR, default "127.0.0.1:8700"

    // Database — DATABASE_URL takes precedence; individual fields used if unset
    DatabaseURL string // env DATABASE_URL  (full postgres DSN)
    DBHost      string // env DB_HOST       default "localhost"
    DBPort      int    // env DB_PORT       default 5432
    DBName      string // env DB_NAME       default "lpwallet"
    DBUser      string // env DB_USER
    DBPassword  string // env DB_PASSWORD
    DBSSLMode   string // env DB_SSL_MODE   default "prefer"

    // TLS
    TLSCertFile string // env TLS_CERT_FILE
    TLSKeyFile  string // env TLS_KEY_FILE

    // Logging
    LogLevel string // env LOG_LEVEL  default "INFO"

    // Seed admin — used only on first boot (no existing admin row)
    AdminEmail    string // env ADMIN_EMAIL     default "admin@localhost"
    AdminPassword string // env ADMIN_PASSWORD  required if seeding
}
```

### DSN resolution

`Config.DSN()` returns the connection string:
- Return `DatabaseURL` as-is if set
- Otherwise build: `"host=<DBHost> port=<DBPort> dbname=<DBName> user=<DBUser> password=<DBPassword> sslmode=<DBSSLMode>"`

### YAML config file

Parsed with `gopkg.in/yaml.v3`. The struct tags on `Config` use `yaml:` keys that match the YAML field names below.

Default path: `config.yaml` in the working directory. Override with `CONFIG_FILE` env var.

If the file does not exist, startup continues using env vars only. If the file exists but cannot be parsed, startup fails.

```yaml
listen_addr: "127.0.0.1:8700"

database:
  url: ""           # full DSN — overrides all fields below if set
  host: "localhost"
  port: 5432
  name: "lpwallet"
  user: ""
  password: ""
  ssl_mode: "prefer"

tls:
  cert_file: ""
  key_file: ""

log_level: "INFO"

admin:
  email: "admin@localhost"
  password: ""
```

### Merge order

`Load()` applies values in this order — later sources win:

1. Hardcoded defaults
2. YAML file (path from `CONFIG_FILE` env var, default `config.yaml`)
3. Environment variables

Only non-empty env var values override the file; an unset env var leaves the file value intact.

### Load rules

- `Load() (*Config, error)` applies the merge order above, then validates
- Error if both `database.url` and all of `database.host`/`database.user`/`database.name` are empty after merging
- Error if one of `tls.cert_file` / `tls.key_file` is set without the other
- `admin.password` is only required when the database has no admin user yet; do not fail startup if absent

## Logging

- Use stdlib `log/slog` — no third-party logging packages
- Initialize once at startup: `slog.New(slog.NewJSONHandler(os.Stdout, nil))`
- Pass `*slog.Logger` via dependency injection alongside `*gorm.DB`
- Default level `INFO`; override with `LOG_LEVEL` env var (`DEBUG`, `INFO`, `WARN`, `ERROR`)
- HTTP middleware logs: `method`, `path`, `status`, `duration_ms`, `request_id`
- Audit callbacks log: `table`, `row_id`, `user_id`, `action` at `DEBUG` level (the DB rows are the authoritative record)

## GORM Audit Plugin

Implement in `internal/db/audit_plugin.go` as a GORM plugin (`plugin.Plugin` interface).

**Audited tables:** `public.user`, `public.account`, `public.account_transaction`

**Action constants** (stored in `audit_event.action` smallint):

| Constant | Value |
|---|---|
| `AuditActionCreate` | `1` |
| `AuditActionUpdate` | `2` |
| `AuditActionDelete` | `3` |

**Registration** — call once at DB init:
```go
db.Use(&db.AuditPlugin{})
```

**Callback registration** — register after the built-in GORM callbacks:
```go
db.Callback().Create().After("gorm:create").Register("audit:after_create", auditAfterWrite(AuditActionCreate))
db.Callback().Update().After("gorm:update").Register("audit:after_update", auditAfterWrite(AuditActionUpdate))
db.Callback().Delete().After("gorm:delete").Register("audit:after_delete", auditAfterWrite(AuditActionDelete))
```

**Inside each callback:**
1. Check `tx.Statement.Table` against the three audited table names; skip all others
2. Read the authenticated user ID from `tx.Statement.Context` using the same context key used by auth middleware
3. Reflect on `tx.Statement.Dest` to extract changed field names and their string representations for `audit_detail` rows
4. Write one `audit_event` row (action, datetime=NOW(), rid_user, row_id, tablename) using `tx` (same transaction)
5. Write one `audit_detail` row per changed column (column_name, column_value, rid_audit_detail=new event ID) using `tx`
6. Log at `DEBUG` level on success; log at `ERROR` level and return the error on failure (which rolls back the parent transaction)

**Context key** — define a single unexported type in `internal/middleware/auth.go` and export only the key value so the audit plugin can import it without a circular dependency.

## Coding Conventions

- No global state; pass `*gorm.DB` and `*slog.Logger` via dependency injection into handlers
- Middleware stack: recovery → logging → auth check → handler
- Do not add third-party router packages; use `net/http` `ServeMux`
- HTTPS: load TLS cert/key from env vars `TLS_CERT_FILE` / `TLS_KEY_FILE`; fall back to HTTP if unset
- Environment variables for config: `DATABASE_URL`, `LISTEN_ADDR` (default `127.0.0.1:8700`), `TLS_CERT_FILE`, `TLS_KEY_FILE`, `LOG_LEVEL` (default `INFO`)
